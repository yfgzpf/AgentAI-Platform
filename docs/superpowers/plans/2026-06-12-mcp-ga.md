## Task 1: 类型 + 5 server 模板

- [ ] 1.1 创建 `mcp/types.ts`, 写 `McpServerConfig` / `OAuth2State` / `McpToolDefinition`
- [ ] 1.2 创建 `mcp/servers.ts`, 写 5 个内置 server 模板(fs/github/slack/supabase/browser), 每个含: id, name, type, default URL, default capabilities, auth type
- [ ] 1.3 写 `mcp/servers.test.ts`: 5 模板字段完整性 + 至少 1 capability

## Task 2: HTTP+SSE client

- [ ] 2.1 创建 `mcp/client.ts`, 实现 `McpClient` 类:
  - `initialize(serverConfig): Promise<{serverInfo, capabilities}>`
  - `listTools(): Promise<McpToolDefinition[]>`
  - `callTool(name, args): Promise<any>`
  - 健康检查 `ping(): Promise<boolean>`(60s 间隔)
- [ ] 2.2 实现 SSE 长连接解析(`text/event-stream` 格式)
- [ ] 2.3 实现重试 + 指数退避(3 次, 1s/2s/4s)
- [ ] 2.4 写 `mcp/client.test.ts`: 用 nock 或自建 http server mock, 跑 initialize + listTools + callTool

## Task 3: OAuth 2.1 PKCE

- [ ] 3.1 创建 `mcp/oauth.ts`, 实现:
  - `generatePkcePair(): {codeVerifier, codeChallenge}`
  - `buildAuthUrl(serverConfig, state): string`
  - `exchangeCodeForToken(serverConfig, code, codeVerifier): Promise<OAuth2State>`
  - `refreshToken(state): Promise<OAuth2State>`
- [ ] 3.2 实现 token 加密存储(用 `keys-cli` 现有 AES-256-GCM)
- [ ] 3.3 写 `mcp/oauth.test.ts`: PKCE 长度 + URL 拼接 + 加密往返

## Task 4: McpManager 单例

- [ ] 4.1 创建 `mcp/index.ts`, `McpManager` 类:
  - 启动时读 `~/.agentai/mcp-servers.json` 加载用户配置
  - `start()`: 对每个 enabled server 异步 initialize
  - `listServers(): McpServerConfig[]`
  - `toggleServer(id, enabled): void`
  - `getAllTools(): McpToolDefinition[]`(聚合所有 server)
  - 60s 健康检查循环
  - `stop()`: 断所有连接
- [ ] 4.2 写 `mcp/index.test.ts`: 用 mock client 测 5 server 启停 + 工具聚合

## Task 5: REST router

- [ ] 5.1 创建 `mcp/router.ts`, `createMcpRouter(manager)`:
  - `GET /v1/mcp/servers` → `{servers: McpServerConfig[]}`
  - `POST /v1/mcp/servers/:id/toggle` body `{enabled: bool}` → `{ok}`
  - `GET /v1/mcp/tools` → `{tools: McpToolDefinition[]}`
  - `POST /v1/mcp/oauth/init` body `{serverId}` → `{authUrl, state}`
  - `GET /v1/mcp/oauth/callback?code&state` → `{ok, tokenExpiresAt}`
- [ ] 5.2 修改 `index.ts`:
  - `const mcpManager = new McpManager(); await mcpManager.start();`
  - `app.use('/v1/mcp', createMcpRouter(mcpManager));`
  - SIGTERM: `mcpManager.stop()`
- [ ] 5.3 写 `mcp/router.test.ts`: 5 端点 + 错误处理

## Task 6: tool-registry 桥接

## 风险与备选

- **风险 1**: `@modelcontextprotocol/sdk` npm 包可能版本不稳定 → 备选: 手写 HTTP+SSE client(已写在 Task 2)
- **风险 2**: OAuth 流程复杂, edge case 多 → 备选: 本季只做 bearer token, OAuth 推迟
- **风险 3**: 5 个内置 server 没人维护 URL → 备选: 全部允许用户自定义 URL, 我们不预设

## 不做(明确)

- ❌ stdio transport(只 HTTP+SSE, 简化)
- ❌ Resource 订阅(只 tool 调用)
- ❌ Prompt template(只 tool)
- ❌ 自建 MCP server
